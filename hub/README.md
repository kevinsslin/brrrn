# brrrn hub

Cloudflare Worker + KV backend for private brrrn pits. It stores daily
aggregates only. Prompts, repository names, paths, and raw sessions never
leave a member's Mac.

## Test

```sh
npm test
```

The suite runs under plain Node with a Map-backed KV mock.

## Local development

```sh
npx wrangler dev --local
```

## Deploy

Create the KV namespace, copy the returned ID into `wrangler.toml`, then
deploy:

```sh
npx wrangler kv namespace create BRRRN_KV
# replace REPLACE_WITH_KV_NAMESPACE_ID in wrangler.toml
npx wrangler deploy
```

Point clients at the printed `workers.dev` URL:

```sh
brrrn config set-hub https://brrrn-hub.<account>.workers.dev
```

## API

All bodies and responses are JSON.

### Create a pit

```http
POST /pit
{"name":"台北燒錢俱樂部"}
```

Returns `{"code":"ember-fox-x7kq"}`. Codes are generated by the server and
checked against KV before creation.

### Join

```http
POST /pit/:code/join
{"handle":"kevin","secret":"client-generated-secret"}
```

Handles are unique within one pit. Only a SHA-256 digest of the secret is
stored.

### Submit daily aggregates

```http
POST /pit/:code/submit
{
  "handle":"kevin",
  "secret":"client-generated-secret",
  "machine_id":"random-machine-id",
  "days":[{
    "date":"2026-07-13",
    "tokens":9655458741,
    "cost_usd":5420.15,
    "claude_usd":113.52,
    "codex_usd":5306.63,
    "models":{
      "gpt-5.6-sol":{"input_tokens":7380000000,"output_tokens":14500000,"cost_usd":4821.87}
    }
  }]
}
```

A machine overwrites its own record for a date. Records from multiple machines
are summed on reads.

### Board

```http
GET /pit/:code/board
```

Returns members sorted by today's USD cost, with UTC today, ISO calendar week,
calendar month, $5/day streak, and model-level input/output details for hover
UI.

### Member history

```http
GET /pit/:code/member/:handle
```

Returns that member's daily token and cost series summed across machines.
